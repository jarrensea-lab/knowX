```
┌─────────────────────────────────────────────┐
│                  数据源层                      │
│  腾讯财经 │ 东方财富 │ 新浪财经 │ (mootdx)       │
└──────────────────┬──────────────────────────┘
                   │
                   ▼

│              多源路由层 (DataRouter)            │
│  首选 → 备选1 → 备选2 → 熔断                     │

└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│           分层缓存 (TieredCache)               │
│  L1 (20s) │ L2 (2min) │ L3 (8min) │ L4 (1h)  │

└──────────────────┬──────────────────────────┘

                   ▼

                   │
                   ▼
┌─────────────────────────────────────────────┐
│              输出分发                          │
│  WebSocket 推送 │ REST API │ 飞书 Webhook      │
└─────────────────────────────────────────────┘
```

## 各层职责

### 数据源层
- **腾讯财经**: 主力数据源，提供实时行情、K线、批量查询
- **东方财富**: 板块排名、涨跌分布、市场宽度
- **新浪财经**: 备用数据源，实时报价
- **mootdx**: 暂不可用（Python 3.14 不兼容），待修复

### 路由层
- 按优先级尝试数据源
- 失败自动切换到备选
- 所有源都失败则熔断，指数退避恢复
- 熔断窗口: 30s → 60s → 120s → 240s → 上限 300s

### 缓存层
- L1: 实时行情 (20s TTL) — 避免重复请求
- L2: 日内数据 (2min TTL) — 板块排名等
- L3: 高频查询 (8min TTL) — 技术指标
- L4: 低频数据 (1h TTL) — 基本面数据
- 淘汰策略: LRU + TTL 过期

### AI 引擎
- 盘前: 全辩论模式（4 角色，~3min）
- 盘中: 快速模式（单次调用，~1min）
- 复盘: 单模型模式（~1min）
- 风险检查: 6 维度加权评分（实时，<1s）

### 输出分发
- **WebSocket**: 实时风险告警推送
- **REST API**: 前端按需查询
- **飞书 Webhook**: 盘前策略 / 风险告警 / 盘中分析 / 复盘总结

## 关键数据流路径

### 路径 1: 盘前分析
```
定时触发 (09:00)
  → 读取活跃持仓 (SQLite)
  → 采集三大指数 (腾讯批量)
  → 采集板块 Top10 (东财)
  → 采集涨跌家数 (东财)
  → 采集持仓实时数据 (腾讯批量)
  → AI 辩论引擎 (4角色, ~3min)
  → 存入 ai_strategies 表
  → 飞书推送 + API 可用
```

### 路径 2: 风险检查
```
定时触发 (5 次/天)
  → 遍历活跃持仓
  → 获取实时数据 (路由 + 缓存)
  → 6 维度评分 (< 1s)
  → 高风险: 存入 risk_alerts + WebSocket 广播 + 飞书推送
  → 中低风险: 静默记录
```

### 路径 3: 盘中分析
```
定时触发 (11:30 / 14:00)
  → 读取持仓 + 计算实时盈亏
  → 采集三大指数 + 板块 Top10
  → 获取最近 10 条风险告警
  → AI 快速分析 (9B, ~60s)
  → 存入 ai_strategies 表
  → 飞书推送
```

### 路径 4: 手动查询
```
用户操作 (前端)
  → GET /api/* (读缓存/数据库)
  → POST /api/*/generate (后台异步生成)
  → 前端轮询 (5s/次, 最长 5min)
  → 轮询成功后刷新 UI
```

## 异步生成与轮询

```
用户点击「生成分析」
  → POST /api/ai/premarket/generate
  → 后端: 立即返回 {"status": "started"}
  → 启动 BackgroundTasks: AI 辩论
  → 前端: pollUntilGenerated()
    → 每 5s 查询 GET /api/ai/premarket
    → 检查 generation_status.running
    → 直到 running=false, 获取数据
    → 最长等待 5min, 超时提示
```
