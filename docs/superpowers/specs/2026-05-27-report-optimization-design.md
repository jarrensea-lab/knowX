# AI 报告推送优化设计

> 盘前 / 盘中 / 复盘报告的飞书推送格式、容错降级、内容结构优化

## 背景

用户反馈三个核心问题：
1. **报告内容不完整**：飞书推送截断 AI 产出的关键信息（推荐股票、具体价位被 `[:120]` 等截断），用户看不到具体投资建议
2. **报告生成成功率不高**：模型超时/返回空/JSON 解析失败导致静默失败，无任何通知
3. **复盘报告质量低**：复盘用单模型 REVIEW_PROMPT，输出远不如盘前的辩论模式

## 方案设计

### 一、飞书推送消息重构

每条分析从「单条卡片截断推送」改为「多卡片独立完整推送」：

| 序号 | 标题 | 内容 |
|------|------|------|
| 1 | 综合决策卡 | 市场判断 + 最终决策 + 信心分 + 仓位建议 + 风险摘要 |
| 2 | 短线策略卡 | 猎手视角摘要 + 持仓操作建议(完整) + 推荐股票(买入区间/止损/目标/新手解读，完整不截断) |
| 3 | 中低频策略卡 | 账房视角摘要 + 持仓操作建议(完整) + 推荐股票(买入区间/止损/目标/新手解读，完整不截断) |
| 4 | 关键数据卡 | 支撑结论的核心数据：指数表现 + 行业资金流 Top5 + 概念资金流 Top5 |

- 盘前报告：4 条推送
- 盘中报告：1-2 条推送（操作建议为主）
- 复盘报告：1 条推送（快速摘要）

每条卡片的 `markdown` 内容控制在 3000 字符以内，但对 AI 产出的关键字段（`reason`、`buy_range`、`stop_loss`、`target`、`beginner_note`）不做字符截断。

### 二、三级降级容错链路

```
正常链路 (辩论模式)
  ├─ 猎手(35B) + 账房(4B) + 守夜人(4B) + 裁判(R1)
  ├─ 各角色 90s 超时
  └─ 任一角色失败 → 一级降级
        ├─ 单模型快速模式 (R1 或主模型, ~60s)
        └─ 失败 → 二级降级
              ├─ 模板化摘要（纯数据，无 AI 分析）
              ├─ 包含：指数行情 + 持仓盈亏 + 行业资金流 Top10
              ├─ 推送仅 1 条（综合数据卡）
              └─ 标注：「⚠️ 降级报告 (L2) — AI 服务暂不可用，以下为客观市场数据」
```

**关键规则**：
- 每个环节超时 90s，超时即降级
- 降级推送必须在标题/正文中标注降级级别（如 `⚠️ 降级报告 (L1)`）
- 盘前走完整链路，盘中保持现有极速模式（2B 单次，不降级），复盘走简化链路
- 手动触发的 API（`/api/ai/*/generate`）同样走降级链路

### 三、三类报告内容优化

#### 盘前报告（保持辩论模式）

**数据输入增强**：
- 已有：行业资金流向 Top10 + 新闻上下文 + 自选行情
- 新增：概念资金流向 Top10 + 沪深港通净流向 + 自选股 K 线简要趋势

**输出对应**：
- 综合决策卡 ← 裁判的 `final_decision` + `reasoning` + `position_advice` + `risk_summary`
- 短线策略卡 ← 裁判的 `short_term`（完整 holdings_advice + recommendations）
- 中低频策略卡 ← 裁判的 `mid_low_freq`（完整 holdings_advice + recommendations）
- 关键数据卡 ← 市场数据采集结果

#### 盘中报告（保持极速模式）

**优化**：
- 输入增加：个股资金流向、大单追踪数据
- 飞书推送 1-2 条，聚焦可执行操作
- 保留「今日一课」新手教育字段
- 盘中失败不降级推送（风险检查和 AI 分析各自独立）

#### 复盘报告（简化为快速摘要）

- 不再走辩论/单模型深度分析
- 直接推送模板化数据摘要：
  - 当日盈亏汇总 + 指数表现 + 行业资金流 Top10
  - 可选：调用一次快速 AI 生成「次日简要展望」段落（30s 超时，失败跳过）
- 当前 `REVIEW_PROMPT` 保留，供手动触发 `/api/ai/review/generate` 时使用

## 实现要点

### 涉及文件

| 文件 | 变更 |
|------|------|
| `backend/app/main.py` | 重构 3 个 job 的推送逻辑；新增降级链路函数；重构 feishu_msg 构建 |
| `backend/app/ai/debate.py` | 可选：优化 `debate_intraday` 输入参数 |

### 关键函数变更

1. **新增** `_send_multi_card(contents: list[dict])` — 批量发送多条飞书卡片
2. **新增** `_build_decision_card(final: dict) -> str` — 构建综合决策卡 markdown
3. **新增** `_build_strategy_card(strategy: dict, label: str) -> str` — 构建策略卡 markdown（短线/中低频）
4. **新增** `_build_data_card(market_data: list, industry_flow: list, concept_flow: list) -> str` — 构建关键数据卡
5. **新增** `_build_degraded_card(level: int, market_data: dict) -> str` — 构建降级报告卡
6. **重构** `premarket_analysis_job()` — 多卡推送 + 降级链路
7. **重构** `review_job()` — 简化为快速摘要推送
8. **优化** `intraday_analysis_job()` — 输入数据增强

### 降级标注格式

```python
DEGRADE_LABELS = {
    1: "⚠️ 降级报告 (L1) — AI 辩论模式失败，切换至快速模式",
    2: "⚠️ 降级报告 (L2) — AI 服务暂不可用，以下为客观市场数据",
}
```

## 不变更的部分

- 调度时间表不变（9:00/11:30/14:00/15:00）
- 数据库 schema 不变
- API 端点不变
- 辩论引擎核心逻辑不变
- 前端不变（本次仅优化后端推送）
