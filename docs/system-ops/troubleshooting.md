# 常见问题排查

> 手动投喂模板 — 记录系统运行中的常见问题和解决方案

---

## 后端问题

### Ollama 连接失败

**现象**: `Connection refused` 或 AI 分析无响应
**排查**:
1. `ollama serve` 是否在运行？→ `curl http://localhost:11434/api/tags`
2. 模型是否已拉取？→ `ollama list`
3. 端口是否被占用？→ `lsof -i :11434`
**解决**: 重启 Ollama 服务

### AI 分析返回空结果

**现象**: AI 分析页面显示空白
**排查**:
1. 检查 AI 模型是否已拉取: `ollama list`
2. 检查 Ollama 日志: `ollama logs`
3. 检查后端日志中的 AI 调用错误
**常见原因**:
- 模型未拉取
- 模型内存不足 (Ollama 自动卸载了模型)
- 提示词中的 `"format": "json"` 参数问题 (v1.0.1 已修复)

### 数据库锁定

**现象**: `database is locked` 错误
**排查**: SQLite 单写锁，多并发写入会锁
**解决**: 
- SQLite 不适合高并发，考虑升级到 PostgreSQL
- 确保数据库连接及时关闭 (`db.close()`)

## 前端问题

### 页面空白

**现象**: 访问 `localhost:8000` 页面空白
**排查**:
1. 前端是否已构建？→ 检查 `frontend/dist/` 目录是否存在
2. 后端是否正确挂载静态文件？
3. 浏览器控制台是否有 JS 错误？
**解决**: `cd frontend && npm run build`

### API 请求超时

**现象**: 前端报 `timeout` 错误
**排查**:
1. 后端是否在运行？→ `curl http://localhost:8000/api/health`
2. 网络是否正常？
3. CORS 是否配置正确？
**解决**: 
- AI 生成类请求超时 10s, 使用轮询机制获取结果
- 其他请求超时 30s

## 数据源问题

### 实时行情不更新

**现象**: 价格数据长时间不变
**排查**:
1. 是否在交易时段？（非交易时段数据不变是正常的）
2. 检查缓存是否过期？
3. 数据源是否熔断？
**解决**: 
- 查看后端日志中的数据源熔断信息
- 手动调用 `/api/health` 查看缓存命中率

### 板块数据为空

**现象**: 板块排名显示为空
**排查**: 东方财富 push2 API 是否可访问？
**解决**: 东财 API 偶尔限流，等待自动恢复

## 环境问题

### Python 版本兼容性

- mootdx 与 Python 3.14 不兼容 (已知限制)
- 推荐 Python 3.11 - 3.13

### macOS 权限问题

**现象**: `Permission denied` 运行脚本
**解决**: `chmod +x scripts/*`
