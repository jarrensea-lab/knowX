## 第8章 趋势跟踪策略

### 核心概念
- **趋势**：上升趋势、下降趋势、横盘趋势
- **移动平均线**：平滑价格走势，确定趋势方向
- **金叉**：短期MA上穿长期MA → 买入信号
- **死叉**：短期MA下穿长期MA → 卖出信号
- **支撑位/阻力位**：MA可作为支撑/阻力参考

### 信号准确度判断原则
1. 交叉角度要锐利
2. 交叉信号要持续一段时间确认
3. 均线斜率要发生显著变化

### 策略实施流程
1. 数据采集和加载
2. 计算移动平均线
3. 制定策略规则（买入/卖出条件）
4. 生成买卖信号
5. DeepSeek辅助模拟回测验证
6. 绘制K线图和信号
7. DeepSeek辅助优化策略（参数调整）

### 关键代码模式
```python
# 计算移动平均线
df['MA5'] = df['Close'].rolling(window=5, min_periods=1).mean()
df['MA20'] = df['Close'].rolling(window=20, min_periods=1).mean()

# 买入信号：短期MA上穿长期MA
def buy_signal(row):
    return row['MA5'] > row['MA20']

# 卖出信号：短期MA下穿长期MA 或 收盘价跌破MA20的92%
def sell_signal(row):
    return row['MA5'] < row['MA20'] or row['Close'] < row['MA20'] * 0.92
```

## 第9章 动量策略与DeepSeek智能辅助决策

### 动量策略概述
基于市场趋势的交易策略，利用资产价格的持续上涨或下跌趋势进行交易。认为过去的价格趋势在一段时间内往往会延续。

### 动量指标
| 指标 | 说明 |
|------|------|
| 动量 | 价格变动速度和幅度 |
| RSI | 相对强弱指标，衡量超买超卖 |
| MA交叉 | 短期/长期MA交叉信号 |
| 通道突破 | 突破价格区间信号 |

### RSI指标
**公式**：`RSI = 100 - 100 / (1 + RS)`，其中 `RS = 上涨日平均收益 / 下跌日平均收益`

**判断标准**：
- `RSI > 70`：超买，可能回调 → 卖出
- `30 < RSI < 70`：行情平稳 → 持有
- `RSI < 30`：超卖，可能反弹 → 买入

### 动量策略优缺点
**优点**：按趋势交易利润大、交易频率高适合短期、指标简单易实施
**缺点**：交易风险大、易受市场噪音影响、长期表现不佳、交易成本高、难以全自动化

### DeepSeek辅助动量策略流程
1. 收集数据
2. 数据预处理
3. 定义策略规则
4. DeepSeek辅助决策
5. 解读回答
6. 优化策略
7. 回测评估

## 第11章 借助DeepSeek构建与优化高频交易策略

### 高频交易（HFT）定义
利用快速执行和高度自动化，通过大量交易机会和微小价格差异实现利润，可在毫秒或微秒级别执行。

### HFT核心概念
- **低延迟**：从接收数据到执行指令时间最短
- **交易算法**：决定何时交易的核心规则
- **流动性**：可供交易的资产数量
- **套利**：利用价格差异获取利润
- **市场制造者**：提供买卖报价增加流动性

### HFT常见算法策略
1. **套利策略**：跨市场、跨品种价格差异
2. **市场制造策略**：从买卖价差中获利
3. **统计套利策略**：利用统计模型的偏差
4. **趋势跟踪策略**：短期/中长期趋势
5. **事件驱动策略**：基于事件或公告

### HFT框架核心组成
1. **市场接入层**：数据源接入
2. **策略引擎**：策略逻辑执行
3. **订单管理系统**：下单、撤单、查询
4. **风险管理层**：账户余额、最大亏损控制
5. **基础设施**：低延迟硬件和网络

### BHTF（基本高频交易框架）

### 套利类型
- **市场套利**：不同市场/交易所价格差异
- **期货套利**：现货与期货价格差异（正/反向套利）
- **无风险套利**：金融工具间价格差异
- **统计套利**：资产价格统计关系
- **跨品种套利**：相关商品间价格差异

### 套利实施步骤
1. 识别套利机会
2. 确定交易策略
3. 市场调研
4. 风险管理
5. 快速执行
6. 跟踪监控
7. 执行交易
8. 盈亏结算

### 重要案例
#### 跨市场套利
股票A在上海交易所100元/股 vs 香港交易所130港币/股，通过汇率换算（1港币=0.9元人民币）实现套利。

#### 配对交易套利（Pairs Trading）
- 原理：基于均值回归，选择两只高相关性股票
- 当价差扩大时：卖出高价资产，买入低价资产
- 当价差回归时：反向操作获利
- 案例：中国石油（601857）与中国石化（600028），皮尔逊相关系数0.933

## 第14章 量化交易回测框架与DeepSeek优化

### 回测定义与作用
用历史数据验证交易策略表现，评估盈利能力、测试稳健性、发现潜在风险。

### 常见回测框架
| 框架 | 特点 |
|------|------|
| **Backtrader** | 灵活模块化Python框架，适合中低频策略 |
| Zipline | 由Quantopian开发，已停止维护 |
| QuantConnect | 云端全栈量化平台 |
| VectorBT | 向量化回测，适合高频参数优化 |
| PyAlgoTrade | 轻量级，适合新手入门 |

### Backtrader核心组件
1. **Cerebro**：回测引擎，协调数据加载、策略执行、结果分析
2. **Strategy**：策略类，定义交易逻辑
3. **Data Feeds**：数据源（CSV、数据库、API）
4. **Indicators**：技术指标（MA、RSI等）
5. **Broker**：模拟经纪人，处理订单执行、佣金、滑点
6. **Analyzers**：分析器（夏普比率、最大回撤）

### Backtrader使用流程
```python
import backtrader as bt

# 1. 定义策略
class MyStrategy(bt.Strategy):
    params = (("short_period", 50), ("long_period", 200))
    def __init__(self):
        self.short_ma = bt.indicators.SMA(self.data.close, period=self.p.short_period)
        self.long_ma = bt.indicators.SMA(self.data.close, period=self.p.long_period)
    def next(self):
        if self.short_ma > self.long_ma and not self.position:
            self.buy()
        elif self.short_ma < self.long_ma and self.position:
            self.sell()

# 2. 配置回测
cerebro = bt.Cerebro()
cerebro.adddata(data)
cerebro.addstrategy(MyStrategy)

# 3. 运行
cerebro.run()
cerebro.plot()

# 4. 参数优化
cerebro.optstrategy(MyStrategy, short_period=range(10, 100, 10))
```

### 止损与止盈策略

| 策略类型 | 说明 | 适用场景 |
|---------|------|---------|
| 固定止损 | 买入价固定比例（如-5%） | 低波动市场/短线 |
| 移动止损（追踪止损） | 止损价随盈利上浮 | 趋势跟踪 |
| 波动性止损 | 基于ATR动态调整 | 高波动市场 |
| 固定止盈 | 买入价固定比例（如+10%） | 短线目标明确 |
| 移动止盈 | 止盈价动态调整 | 趋势跟踪 |
| 技术指标止盈 | 基于RSI/布林带 | 中长线 |
| 固定止损+固定止盈 | 同时设置 | 综合风控 |
| 移动止损+移动止盈 | 同时动态调整 | 综合风控 |
